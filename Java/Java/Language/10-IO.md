# Input and Output

## Input/Output Streams

### Byte Streams (`InputStream` / `OutputStream`)

- `InputStream.read()` — returns byte as `int` (0–255) or `-1` at end
- `InputStream.readAllBytes()` — read entire stream into `byte[]`
- `InputStream.transferTo(out)` — copy all bytes from input to output
- `OutputStream.write(int b)` — write one byte
- `OutputStream.write(byte[])` — write entire array
- Both `read` and `write` __block__ until data is available
- `available()` — number of bytes readable without blocking
- Always `close()` streams — frees OS resources; closing output also flushes buffer
- `flush()` — manually flush buffered output
- `InputStream.nullInputStream()` / `OutputStream.nullOutputStream()` — discard sinks

### Reader/Writer Hierarchy

- `Reader.read()` — returns UTF-16 code unit (0–65535) or `-1`
- `Writer.write(int c)` — writes a Unicode code unit
- `Closeable` extends `AutoCloseable` — use try-with-resources; `close()` only throws `IOException` (vs. `AutoCloseable` which can throw any)
- `Flushable` — implemented by `OutputStream` and `Writer`
- `Readable` — `int read(CharBuffer cb)` — implemented by `Reader`
- `Appendable` — `append(char)` / `append(CharSequence)` — implemented by `Writer`

### Combining Stream Filters

- Compose streams by nesting constructors:

```java
var din = new DataInputStream(
    new BufferedInputStream(
        new FileInputStream("employee.dat")));
```

- `BufferedInputStream` / `BufferedOutputStream` — block-level buffering
- `DataInputStream` / `DataOutputStream` — read/write primitive types in binary
- `PushbackInputStream` — `unread(b)` to push back one byte; buffer default length 1
- Nesting order matters: outermost class determines which methods are available

### Text Input/Output

- `InputStreamReader` / `OutputStreamWriter` — bridge between byte streams and character streams
- Since Java 18: default encoding is UTF-8; before Java 18, was platform encoding
- Specify encoding explicitly for portable code: `new InputStreamReader(in, StandardCharsets.UTF_8)`

### Reading Text

```java
String content = Files.readString(path);
List<String> lines = Files.readAllLines(path);
try (Stream<String> lines = Files.lines(path)) { ... }
Scanner in = new Scanner(path);
in.useDelimiter("\\PL+");  // custom delimiter regex
Stream<String> tokens = in.tokens();
```

- `Scanner.nextLine()` / `nextInt()` / `nextDouble()` — consume tokens; `hasNextLine()` / `hasNextInt()` to check
- `nextInt()` does NOT consume the trailing newline — call `nextLine()` after to discard it
- `Scanner` from `Path` uses platform encoding by default

### Writing Text

```java
var out = new PrintWriter("employee.txt", StandardCharsets.UTF_8);
out.print(name); out.println(salary);
out.printf("%s: %.2f%n", name, salary);
out.checkError();  // true if any error occurred (no exceptions thrown)
```

- `println` adds platform line separator (`System.getProperty("line.separator")`)
- Enable autoflush: `new PrintWriter(writer, true)` — flushes on every `println`/`printf`
- Cannot construct `PrintWriter` from a `Path`

### Character Encodings

- UTF-8: variable 1–4 bytes; ASCII is 1 byte; most common on the Internet
- UTF-16: Java's internal encoding; 1 or 2 16-bit values; big/little endian variants
- BOM (0xFEFF) indicates byte order; UTF-8 BOM is unnecessary but allowed — Java does NOT strip it
- `StandardCharsets.UTF_8`, `UTF_16`, `ISO_8859_1`, `US_ASCII`, etc.
- `Charset.forName("Shift-JIS")` — get charset by name
- `Charset.defaultCharset()` — current default

### Default Encoding Behaviour

- `Files.*` methods: UTF-8 default
- `java.io` readers/writers, `String(byte[])`, `Scanner`, `Formatter`, `URLEncoder`: use default charset
- Set default charset via `-Djava.util.logging.config.file` → actually `-Dfile.encoding=UTF-8`
- Java 25: `stdin.encoding` property for `System.in` reading
- Typical Windows vs. Linux: `native.encoding` = Cp1252 vs. UTF-8; `stdout.encoding` = cp437 vs. UTF-8

## Binary Data

### `DataInput` / `DataOutput`

```java
var din = new DataInputStream(new FileInputStream("data.dat"));
var dout = new DataOutputStream(new FileOutputStream("data.dat"));
```

- Methods: `readInt`/`writeInt`, `readDouble`/`writeDouble`, `readBoolean`/`writeBoolean`, `readUTF`/`writeUTF`, `readChar`/`writeChar`, etc.
- All values written in __big-endian__ format — platform independent
- `writeUTF` / `readUTF` — "modified UTF-8" (UTF-16 first then UTF-8 rules); only for Java-to-Java; use `writeChars` for external programs
- `writeLong` (8 bytes), `writeInt` (4), `writeShort` (2), `writeByte` (1), `writeDouble` (8), `writeFloat` (4)

### `RandomAccessFile`

```java
var inOut = new RandomAccessFile("employee.dat", "rw"); // "r", "rw", "rwd", "rws"
inOut.seek(pos);             // position file pointer (long)
long pos = inOut.getFilePointer();
long len = inOut.length();   // total bytes
```

- Implements both `DataInput` and `DataOutput`
- Modes `rwd`/`rws` flush data/metadata to disk synchronously — prevents data loss on crash
- Fixed-size records enable efficient random access: `seek((n-1) * RECORD_SIZE)`

### ZIP Archives

```java
var zin = new ZipInputStream(new FileInputStream("archive.zip"));
ZipEntry entry = zin.getNextEntry();
while (entry != null) {
    // read from zin
    zin.closeEntry();
    entry = zin.getNextEntry();
}
zin.close();
```

- `ZipOutputStream` — `putNextEntry(new ZipEntry(name))` → write bytes → `closeEntry()`
- `setLevel(Deflater.BEST_COMPRESSION)`, `setMethod(DEFLATED | STORED)`
- JAR files = ZIP + manifest; use `JarInputStream`/`JarOutputStream`

## Memory-Mapped Files

```java
try (FileChannel channel = FileChannel.open(path)) {
    MappedByteBuffer buffer = channel.map(
        FileChannel.MapMode.READ_ONLY, 0, channel.size());
    while (buffer.hasRemaining()) {
        byte b = buffer.get();
        ...
    }
}
```

- Modes: `READ_ONLY`, `READ_WRITE`, `PRIVATE`
- Much faster than buffered streams for large files (O(1) vs. O(n) I/O calls)
- `buffer.get(i)` — random access; `buffer.get()` — sequential; `buffer.getInt()`, `buffer.getDouble()`, etc.
- `buffer.order(ByteOrder.LITTLE_ENDIAN)` — change byte order
- `buffer.limit()`, `buffer.hasRemaining()`, `buffer.remaining()`

### Buffer Lifecycle

- Fill: `put()` or channel read → flip → consume: `get()` or channel write → compact → fill again
- `flip()` — limit = position, position = 0 (ready to read)
- `rewind()` — position = 0, limit unchanged (reread)
- `clear()` — position = 0, limit = capacity (discard, ready for new fill)
- `compact()` — copy unconsumed to start, position = after copied, limit = capacity

## File Locking

```java
try (FileChannel channel = FileChannel.open(path, StandardOpenOption.READ, StandardOpenOption.WRITE)) {
    try (FileLock _ = channel.lock()) {   // exclusive, blocking
        // or channel.tryLock() for non-blocking (returns null if unavailable)
        // or channel.lock(start, size, shared) for range locking
        ...
    }
}
```

- `shared = true` — multiple readers allowed; `shared = false` — exclusive
- `lock.isShared()` — check actual lock type granted
- Use `Long.MAX_VALUE` as size to lock all bytes including future growth
- File locking is JVM-global — same thread group shares lock namespace; `OverlappingFileLockException` if overlapping locks in same JVM
- System-dependent: advisory on Linux (other processes may ignore), network file systems unreliable

## Object Serialization

### Basics

```java
var out = new ObjectOutputStream(new FileOutputStream("data.ser"));
out.writeObject(employee);

var in = new ObjectInputStream(new FileInputStream("data.ser"));
Employee e = (Employee) in.readObject();
```

- Class must implement `Serializable` (marker interface, no methods required)
- Fields of type `Serializable` are recursively serialized
- Object graph maintained: shared references preserved; each object assigned a __serial number__
- Repeated references written as serial number back-references

### `transient` Fields

- `transient` fields are skipped during serialization — use for non-serializable types, caches, connections
- Always mark fields as `transient` if they hold non-serializable instances

### Serialization File Format

- Magic: `AC ED 00 05`
- Class descriptors include: class name, 8-byte SHA-1 fingerprint, field names and types
- Repeated class/object references use serial numbers
- `writeObject` / `readObject` private methods can customize serialization
- Static inner classes, enumerations, records are safe to serialize

### Custom Serialization

```java
@Serial private void writeObject(ObjectOutputStream out) throws IOException {
    out.defaultWriteObject();   // write non-transient fields
    out.writeDouble(point.getX());
}

@Serial private void readObject(ObjectInputStream in)
        throws IOException, ClassNotFoundException {
    in.defaultReadObject();
    double x = in.readDouble();
    point = new Point2D.Double(x, in.readDouble());
}
```

- `@Serial` annotation enables IDE/compiler checking of serialization methods
- Fields defined by `serialPersistentFields` override default field list

### `Externalizable`

- Implement `readExternal(ObjectInput)` and `writeExternal(ObjectOutput)` — fully manual
- Must have a no-argument constructor
- Methods are `public` — expose state mutation risk
- Superclass data must be saved/restored explicitly

### `readResolve` / `writeReplace`

- `writeReplace()` — return a proxy object to serialize instead of `this`
- `readResolve()` on the proxy — return the real object (e.g., look up from database/singleton pool)
- Enumerations handle serialization automatically (always deserializes to the correct singleton)
- Records use canonical constructor for deserialization

### Versioning

```java
@Serial public static final long serialVersionUID = 8367346051156850807L;
```

- Without `serialVersionUID`, fingerprint computed from class structure — changes break deserialization
- Add `serialVersionUID` to allow controlled evolution
- Added fields: set to zero/null/false; removed fields: ignored
- Changed field types: incompatible — deserialization fails
- Get original UID: `serialver ClassName`
- `@Serial private void readObjectNoData()` — called when superclass data absent during deserialization

### Security

- Deserialization bypasses constructors — attack vector for crafting invalid objects
- Implement `ObjectInputValidation.validateObject()` + register with `in.registerValidation(this, 0)` in `readObject`
- Use JEP 290/415 serialization filters: `-Djdk.serialFilter='pkg.*;java.**;!*'`
- Avoid deserializing data from untrusted sources; sign session data

### Clone via Serialization

```java
var out = new ByteArrayOutputStream();
new ObjectOutputStream(out).writeObject(obj);
var clone = new ObjectInputStream(new ByteArrayInputStream(out.toByteArray())).readObject();
```

- Deep clone for free; much slower than explicit `clone()`
- Warning: cyclic object graphs with custom `readObject` can fail (ArrayList–HashSet cycle example)

## Regular Expressions

### Key Syntax

| Pattern | Description |
|---|---|
| `.` | Any character (except line terminators by default) |
| `[abc]`, `[a-z]`, `[^0-9]` | Character class, range, complement |
| `\d`, `\D` | Digits / non-digits |
| `\w`, `\W` | Word characters / non-word |
| `\s`, `\S` | Whitespace / non-whitespace |
| `\p{L}`, `\p{Sc}` | Unicode letter / currency symbol |
| `*`, `+`, `?` | 0+, 1+, 0 or 1 repetitions |
| `{n}`, `{n,}`, `{m,n}` | Exact, at least, range repetitions |
| `X?`, `X*?`, `X+?` | Reluctant (shortest match first) |
| `X?+`, `X*+`, `X++` | Possessive (no backtracking) |
| `^`, `$` | Start/end of input (or line in MULTILINE mode) |
| `\b`, `\B` | Word boundary / non-boundary |
| `\b{g}` | Grapheme cluster boundary |
| `(X)` | Capturing group |
| `(?:X)` | Non-capturing group |
| `(?<name>X)` | Named capturing group |
| `\1`, `\k<name>` | Back-reference |
| `\Q...\E` | Literal string (escape all special chars) |
| `X\|Y` | Alternation |
| `(?i:X)` | Case-insensitive match for X |
| `#...` (with COMMENTS flag) | Comment to end of line |

### Testing and Finding

```java
Pattern pattern = Pattern.compile(regex, Pattern.CASE_INSENSITIVE | Pattern.UNICODE_CHARACTER_CLASS);
Matcher matcher = pattern.matcher(input);

matcher.matches()           // entire input matches
matcher.lookingAt()         // input starts with match
matcher.find()              // find next match
matcher.hasMatch()          // true if currently on a match

// Groups
String g = matcher.group(1);      // by index (1-based)
String g = matcher.group("name"); // named group
int start = matcher.start("name");
int end = matcher.end("name");
Map<String, Integer> namedGroups = matcher.namedGroups();
```

### Streams of Matches

```java
List<String> matches = pattern.matcher(input)
    .results()
    .map(MatchResult::group)
    .toList();

Scanner in = new Scanner(path);
Stream<MatchResult> results = in.findAll("\\pL+");
```

### Replace

```java
matcher.replaceAll(",")                           // replace all
matcher.replaceFirst(",")                         // replace first
matcher.replaceAll(m -> m.group().toUpperCase())  // function-based replacement
"3:45".replaceAll("(\\d+):(?<m>\\d{2})", "$1 hours and ${m} minutes")
```

### Split

```java
String[] tokens = pattern.split(input);
String[] tokens = pattern.split(input, limit);      // limit application count
String[] tokens = pattern.splitWithDelimiters(input, -1); // keep delimiters
Stream<String> tokens = pattern.splitAsStream(input);
```

- `split(".")` splits on EVERY character — escape: `split("\\.")` or `split(Pattern.quote("."))`

### Flags

| Flag | Description |
|---|---|
| `CASE_INSENSITIVE` / `i` | Case-insensitive matching (ASCII only) |
| `UNICODE_CASE` / `u` | Case-insensitive for Unicode |
| `UNICODE_CHARACTER_CLASS` / `U` | Use Unicode character classes (implies `u`) |
| `MULTILINE` / `m` | `^`/`$` match line boundaries |
| `DOTALL` / `s` | `.` matches line terminators |
| `COMMENTS` / `x` | Whitespace and `#` comments ignored |
| `LITERAL` | Pattern treated as literal string |
| `CANON_EQ` | Canonical Unicode equivalence |

### Predicate Conversion

```java
Pattern p = Pattern.compile("[0-9]+");
strings.stream().filter(p.asMatchPredicate()).toList() // full match
strings.stream().filter(p.asPredicate()).toList()      // contains match
```
